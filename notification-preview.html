Notification System

A professional, persistent notification bell. Unread state survives page reloads and site redeployments. Read items stay read — they don't come back.

Integration — 3 steps
Step 01

Copy the file

Place notifications.js alongside your HTML pages.

Step 02

Add one script tag

Paste it just before </body> on every page, after Firebase.

Step 03

Zero config

The bell auto-injects into your <nav>. Read state persists across reloads.

Add to each page — after Firebase SDKs 
<!-- StudyHub Notifications -->
<script src="notifications.js"></script>
What's included

Persistent read state — stores read item IDs in localStorage, not a timestamp. Read items never reappear as unread after a reload or redeployment.

Live Firestore listener — badge and list update in real-time when you upload from the Admin Panel. No polling, no refresh needed.

Smooth panel animation — opens with a clean CSS transition (no bounce, no cartoon effects). Closes on outside click or Escape.

Three notification types — study materials (PDFs), friend requests, and profile views, each rendered in their own section with appropriate icons.

Auto-inject, zero config — finds .nav-right or #bellSlot automatically. Works across all pages with a single script tag.
